# Debug Journal — Mermaid Pie Chart Rendering

## Bug
Mermaid pie charts render as raw code text (or fallback `<pre class="mermaid-fallback">`) in Firefox extension instead of SVG.

## Environment
- Runtime: Firefox MV3 content script
- Mermaid bundle: v11.15.0 (esbuild, `globalThis["mermaid"]`)
- Marked: v15.0.12
- Extension loads: `marked.min.js` → `mermaid.min.js` → `content/render.js`

## Hypotheses (Phase 2)

### H1: marked.js doesn't produce `code.language-mermaid` for pie chart syntax
- Pie chart uses ` ```mermaid ` fences → marked should produce `<pre><code class="language-mermaid">`
- If marked treats it as unknown code block (no language), it won't get picked up by `querySelectorAll('code.language-mermaid')`

### H2: The mermaid global is undefined at render time
- `mermaid.version` on line 101 would throw → caught by outer catch in `render()`, shows error page
- esbuild bundle uses `globalThis` — might not work in FF content script depending on CSP/world

### H3: `mermaid.render()` succeeds but returns unexpected value
- `{ svg }` destructuring gets `undefined` → `div.innerHTML = undefined` → empty div, not raw text
- Actually we already fixed this to accept both forms

### H4: `mermaid.render()` throws for pie chart specifically
- Catch block produces red-dashed `<pre class="mermaid-fallback">` with raw source
- User may describe this as "raw code text"

### H5: Content script timing issue — page DOM cleared before mermaid elements exist
- `document.body.innerHTML = ''` on line 153 destroys the text viewer's DOM
- Then `renderMermaidDiagrams` is called — mermaid elements added to container
- But fallback behavior might trigger if DOM isn't ready

## Fixes Applied (in render.js)
- Added `typeof mermaid === 'undefined'` guard with graceful fallback
- Wrapped `mermaid.version` in try/catch
- Accept both `{svg}` object and raw string from `mermaid.render()`
- Added detailed console logging (svg length, result type, stack traces)
- Increased `mermaid.initialize()` timeout from 100ms → 300ms
